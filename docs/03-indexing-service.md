# Сервис индексации (Index Service)

Этот сервис отвечает за то, как данные будут "видны" поисковому движку. Мы не просто индексируем текст, мы создаем **семантический контекст**.

## 1. Обогащение сообщения (Enrichment)
Каждое сообщение перед индексацией превращается в структурированный блок. Это позволяет поиску "зацепиться" за метаданные.

**Пример трансформации:**
*Исходное сообщение:* "Пришлите плз доку по VPN" от `user@vk.com` в чате `IT-Support`.

*Результат для индекса:*
```text
chat: IT-Support type: group
author: user@vk.com
date: 2026-02-20
Пришлите плз доку по VPN
```

## 2. Алгоритм чанкинга (Chunking)
Мы используем метод **скользящего окна (Sliding Window)**:
*   **Размер окна (Chunk Size)**: 512 символов.
*   **Перекрытие (Overlap)**: 128 символов.

**Почему это важно?**
Перекрытие гарантирует, что если важная мысль была разорвана на границе чанка, она полностью попадет в следующий чанк, и связь не потеряется.

## 3. Sparse-вектора (BM25)
Сервис локально вычисляет разреженные вектора. Это "классический" поиск по словам, который отлично находит специфические термины (например, названия серверов `srv-01-prod` или хеш-коды), которые нейросеть может не знать.

## Параметры конфигурации
*   `MAX_CHARS`: 5000 (ограничение на длину входа для моделей).
*   `UVICORN_WORKERS`: 8 (высокая параллельность при обработке больших батчей сообщений).
