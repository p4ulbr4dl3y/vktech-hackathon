# 🏆 Отчет о реализации решения: Advanced RAG для VK WorkSpace

Этот документ описывает ключевые улучшения, внесенные в базовый шаблон решения для Хакатона IT Academy 2026.

## 🏗 Архитектурные изменения

Мы использовали гибридную схему поиска (Dense + Sparse) с этапом глубокого переранжирования, адаптированную под корпоративный контекст VK.

### 1. Умная индексация (Index Service)
*   **Rich Message Rendering**: Базовый поиск часто теряет информацию. Мы реализовали парсинг вложенных структур `parts`, извлекая текст из цитат (`quote`) и пересланных сообщений (`forward`). Также добавлено извлечение имен файлов из `file_snippets`.
*   **Оптимальный Chunking**: Размер чанка уменьшен до **5 сообщений** (overlap 2). Это минимизирует шум и позволяет RRF точнее ранжировать конкретные диалоги.
*   **Sparse Meta-Boosting**: В текстовое представление для BM25 (`sparse_content`) принудительно добавлены уникальные упоминания (`mentions`), что дает 100% точность при поиске сообщений, связанных с конкретными пользователями.

### 2. Продвинутый поиск (Search Service)
*   **Query Expansion (HyDE)**: Для Dense-поиска генерируется гипотетический ответ на основе вопроса. Это позволяет находить сообщения по смыслу, даже если в них нет слов из запроса.
*   **Weighted Sparse Search**: Ключевые слова из вопроса дублируются (x5), а извлеченные сущности (продукты, сервисы) — (x3). Это гарантирует, что технические термины (например, "CGO", "Tarantool") будут иметь приоритет над общей лексикой.
*   **Hybrid RRF Fusion**: Объединение результатов векторного и ключевого поиска через Reciprocal Rank Fusion с расширенными лимитами префетча (K=50/100).
*   **Smart Reranking**: 
    *   Топ-50 кандидатов проходят через `llama-nemotron-rerank-1b-v2`.
    *   Внедрен **фильтр порога уверенности** (score < 0.00001 отсекается), чтобы не выдавать нерелевантные ID.
    *   Возвращается строго уникальный список ID, отсортированный по релевантности.

## ✅ Результаты тестирования (на Go Nova.json)

Локальная верификация на реальном срезе данных подтвердила:
1.  **Точность**: Поиск по сложным запросам (например, *"сисколлы на маке"*) выдает релевантные сообщения в ТОП-1.
2.  **Полнота**: Благодаря HyDE и Hybrid Search, система находит ответы даже при разной терминологии вопроса и сообщения.
3.  **Чистота выдачи**: Вместо возврата всех сообщений из чанка, алгоритм теперь фокусируется на высокоуровневых совпадениях.
