# Поисковый сервис (Search Service)

Это "мозг" системы, использующий ансамбль моделей для достижения рекордной точности.

## Двухстадийный пайплайн

### Стадия 1: Гибридный поиск (Retrieval)
Мы запускаем одновременно 4 потока поиска в Qdrant:
1.  **Dense Query**: Семантический поиск по самому вопросу.
2.  **HyDE (Hypothetical Document Embeddings)**: Поиск по гипотетическому ответу (генерируется заранее).
3.  **Sparse Main**: Поиск по ключевым словам вопроса + ID автора.
4.  **Sparse Optimized**: Поиск по нормализованному "search_text".

Результаты объединяются через **RRF**, что позволяет сбалансировать точность терминов и общий смысл.

### Стадия 2: Реранкинг (Reranking)
Топ-35 кандидатов отправляются в модель **Cross-Encoder (Llama-Nemotron)**. В отличие от обычных векторов, эта модель попарно сравнивает вопрос и каждый найденный текст, вычисляя глубокую вероятность соответствия.

## Эвристический бустинг (Heuristic Boost)
После получения скора от нейросети, мы применяем дополнительные веса:
*   **Entity Match (x1.2)**: Если в тексте чанка есть название документа или ссылка, упомянутая в вопросе.
*   **Date Match (x1.15)**: Если даты в сообщении совпадают с временным интервалом вопроса.
*   **Author Match (x1.1)**: Если сообщение написано автором вопроса (поиск своих сообщений).

## Диверсификация результатов
Алгоритм выбирает уникальные `message_ids`, гарантируя, что в топ-50 попадут только самые релевантные сообщения, исключая дубликаты из перекрывающихся чанков.
